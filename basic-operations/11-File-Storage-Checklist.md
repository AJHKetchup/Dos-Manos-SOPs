# File Storage Checklist

Status: Draft helper checklist
Last updated: 2026-07-03

Use with `01-File-Storage-and-Drive-Handling-SOP.md`.

## Before routing a file

- [ ] Identify the source and document type.
- [ ] Decide whether it is public-safe, internal, customer-sensitive, or restricted.
- [ ] Check whether it belongs in Drive, Agent HQ, SOP repo, Supermove, QuickBooks/accounting, or restricted storage.
- [ ] Preserve the original file and metadata.

## Common destinations

- [ ] Brand board/logo/approved visual rules -> Drive `Brand Source of Truth`.
- [ ] Marketing docs/campaign drafts -> Drive `Brand and Marketing`.
- [ ] Sales scripts/quote resources -> Drive `Sales and Customer Growth`.
- [ ] Affiliate/referral docs/partner CRM -> Drive `Partners and Affiliates`, subject to restricted handling.
- [ ] Strategy/planning -> Drive `Start Here`.
- [ ] Agent reports -> Drive `Agent and Automation HQ` plus Agent HQ source where applicable.
- [ ] SOP markdown source -> SOP repo.
- [ ] Automation/manifests/scripts -> Agent HQ repo.
- [ ] Unsure/duplicate/needs-review -> Drive `Inbox - Needs Sorting`.
- [ ] Finance/HR/legal/ICC/compliance/claims/insurance-sensitive -> restricted branch only.

## Do not do without approval

- [ ] Delete/trash/permanently delete.
- [ ] Change sharing, permissions, ownership, or public access.
- [ ] Move restricted finance, HR, legal, ICC, insurance, compliance, claim, or customer-sensitive files.
- [ ] Rename shared folders into numbered or agent-centric labels.
- [ ] Copy raw sensitive details into GitHub, broad prompts, or public docs.
